## 2) 기능 상세 목록

### A. 기본 UX 개선 기능 (사용자 제안)

#### 1) 🌙 다크/라이트 모드 전환 (난이도 ⭐)
- **설명**: 사용자가 테마를 버튼으로 전환
- **UI**: 헤더 우측 토글(🌙/☀️), 설정 페이지에서도 변경 가능
- **저장 방식**
  - 1차: LocalStorage(즉시 적용, 구현 쉬움)
  - 선택: 사용자 계정 설정(DB) 저장 (로그인 시 기기 간 동기화)
- **체크포인트**
  - 테이블/모달/토스트/폼의 대비(contrast) 확인
  - 프린트(PDF)와 테마 충돌 방지(프린트는 항상 라이트 추천)

#### 2) 🔍 전역 검색 (난이도 ⭐⭐)
- **설명**: 학생/반/강사 등을 한 번에 검색
- **UI**
  - 상단 고정 검색창(또는 `/` 단축키로 포커스)
  - 결과: 카테고리별 섹션(학생/반/강사/수납 등) + 바로가기
  - “최근 검색” 및 “즐겨찾기 결과” 옵션
- **검색 대상 예시**
  - 학생: 이름/전화/학부모 전화/메모/등록코드
  - 반: 반명/시간/강사명
  - 강사: 이름/전화
- **체크포인트**
  - 디바운스(300~500ms) 적용
  - 결과 10~20개 제한 + “더보기” 링크 제공
  - 개인정보(전화/메모) 노출 범위 권한 체크

#### 3) 📤 대시보드 PDF 내보내기 (난이도 ⭐⭐)
- **설명**: 대시보드 현황을 PDF로 저장/공유
- **방식**
  - 서버 렌더링(권장): 템플릿(HTML) → PDF 변환
  - 클라이언트 렌더링: 화면 캡처 기반(PDF 품질/폰트 이슈 가능)
- **포함 항목 예시**
  - 기간(이번 달/분기) 매출/미납/출석률 요약
  - TOP 5 미납자/결석자 리스트
  - 간단 차트(선/막대) 또는 표 중심
- **체크포인트**
  - 인쇄용 CSS(프린트 레이아웃) 분리
  - 페이지 넘김(표 끊김) 처리

#### 4) 📱 모바일 최적화 (난이도 ⭐⭐)
- **설명**: 출결 등 모바일에서 자주 쓰는 기능 개선
- **UI**
  - 하단 탭바(출결/검색/수납/설정)
  - 큰 터치 영역(버튼/행 높이), 스와이프 액션 옵션
  - QR 출결 “원터치 스캔 진입” + 진동/사운드 피드백(선택)
- **체크포인트**
  - 표(Table)는 모바일에서 카드뷰로 전환(권장)
  - 키보드 올라올 때 레이아웃 깨짐 방지

### B. 추가 추천 UX 기능 (체감 좋은 것 위주)

#### 5) 📌 테이블 UX 강화 (난이도 ⭐⭐)
- **포함**
  - 컬럼 정렬/필터/페이지 크기 변경
  - 열 숨기기(가독성) / 컬럼 고정(선택)
  - 빈 결과/로딩/에러 상태 UI 통일
- **체크포인트**
  - 서버 페이징 기반으로 확장 가능하게 설계

#### 6) 🗂️ 저장된 필터(내 필터) (난이도 ⭐⭐⭐)
- **설명**: “미납만”, “이번 주 결석자” 같은 필터를 저장/재사용
- **저장 방식**
  - 1차: 사용자별 DB 저장(추천)
  - 단순: LocalStorage 저장(기기별)
- **UI**: 필터 드롭다운에 “내 필터” 섹션 + 저장/삭제 버튼

#### 7) ↩️ Undo(실행 취소) + 삭제 확인 강화 (난이도 ⭐⭐)
- **설명**
  - 삭제/수정 직후 5~10초 토스트로 “실행 취소”
  - 삭제 모달에 영향 표시(예: “출결 120건, 수납 8건 연결됨”)
- **구현 팁**
  - Soft delete(비활성화) + 복구 지원하면 안정적
  - Hard delete는 관리자만 허용하는 방식도 고려

#### 8) 🧭 목록 상태 유지 (난이도 ⭐⭐⭐)
- **설명**: 목록 → 상세 → 뒤로가기 시 필터/페이지/스크롤 유지
- **방법**
  - 쿼리스트링 유지 + session/localStorage로 스크롤 위치 저장
  - SPA가 아니라도 충분히 구현 가능

#### 9) 🔔 토스트/로딩 UI 통일 + ⏳ 스켈레톤 로딩 (난이도 ⭐⭐~⭐⭐⭐)
- **설명**: 사용자 피드백을 일관된 컴포넌트로 통일
- **권장 규칙**
  - 성공: 2초, 경고/실패: 4초
  - 저장/삭제 등 주요 액션은 버튼 disabled + 스피너 표시

#### 10) 🕘 최근 사용 / ⭐ 즐겨찾기 (난이도 ⭐⭐)
- **설명**: “최근 본 학생”, “즐겨찾는 메뉴/학생” 상단 제공
- **저장 방식**
  - 최근 사용: LocalStorage(간단) / 또는 DB(다기기)
  - 즐겨찾기: DB(권장)

#### 11) ⌨️ 단축키 지원 (난이도 ⭐)
- **예시**
  - `/` 전역 검색 포커스
  - `N` 새 학생 등록
  - `A` 출결 화면 이동
- **주의**: 입력창 focus 상태에서는 단축키 무시

#### 12) 👤 학생 프로필 한 화면 요약(탭) (난이도 ⭐⭐⭐)
- **설명**: 학생 상세에서 출결/수납/상담/성적을 탭으로 통합
- **효과**: 화면 이동 감소, 상담/관리 속도 개선

## 3) 권장 구현 순서 (마일스톤)

### M1 (1~2주)
- 🌙 테마 토글 + 저장
- 🔔 토스트/로딩 UI 통일
- 📌 테이블 정렬/필터/페이지 기본

### M2 (1~2주)
- 🔍 전역 검색(학생/반/강사) + 최근검색
- 📱 모바일 핵심 화면 개선(출결/검색/수납)

### M3 (1~2주)
- 📤 PDF 내보내기(대시보드) + 인쇄 CSS
- ↩️ Undo + 삭제 확인 강화(영향 표시)

### M4 (2주+)
- 🗂️ 저장된 필터(내 필터)
- 🧭 목록 상태 유지
- 👤 학생 프로필 탭 통합

## 4) 기술/구현 가이드 (Django + Bootstrap 기준)

### 공통 원칙
- 서버는 **페이지네이션/필터/검색**을 기본 지원(데이터 많아짐 대비)
- 프론트는 **최소 JS**로도 동작하도록(점진적 향상) 설계
- 개인정보(연락처/메모)는 권한별 노출 제어

### 추천 패키지/옵션 (예시)
- PDF: `WeasyPrint` 또는 `wkhtmltopdf` 계열(서버 렌더링 기반)
- 테이블/필터 UI: Bootstrap + 간단 JS(또는 DataTables를 제한적으로)
- 토스트: Bootstrap Toast 컴포넌트 활용

## 5) 데이터 저장(선택) 설계 예시

### UserPreference (선택)
- theme: `light|dark|system`
- table_page_size: 10/20/50
- mobile_tab_enabled: bool

### SavedFilter (선택)
- user_id
- target: `students|payments|attendance|classes`
- name
- query_json (필터 조건 JSON)

### Favorites / Recent (선택)
- favorites: DB 권장
- recent: LocalStorage 또는 DB(최근 N개만)
